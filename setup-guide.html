⏱ 예상 소요 시간: 약 10~15분
💻 필요한 것: 컴퓨터 + 이메일 주소
💰 비용: GitHub·Vercel 모두 무료

내용 수정
1
GitHub 계정 만들기
GitHub는 내 홈페이지 파일이 저장되는 공간입니다. 무료이고, 한 번 만들면 평생 씁니다.
1-1
GitHub 가입 페이지 접속

아래 버튼을 클릭하거나 브라우저에서 github.com/signup 을 입력하세요.

🔗 GitHub 가입 페이지 열기
💡 이미 GitHub 계정이 있다면 1단계는 건너뛰고 2단계로 이동하세요.
1-2
이메일 · 비밀번호 · 사용자명 입력
  1. 자주 쓰는 이메일 주소 입력
  2. 비밀번호 설정 (영문+숫자 조합 8자 이상)
  3. 사용자명(username) 입력 — 영문·숫자·하이픈(-) 만 사용 가능
⚠️ 사용자명은 나중에 내 저장소 주소에 들어갑니다. 기억하기 쉬운 이름으로 정하세요. (예: myname, john-kim)
1-3
이메일 인증 완료

가입 후 입력한 이메일로 인증 메일이 옵니다.
메일을 열고 "Verify email address" 버튼을 클릭하세요.

✅ 인증 완료 후 GitHub 메인 화면이 보이면 1단계 끝!
2
ZeroPack 로그인 & 라이선스 키 입력
크몽에서 구매한 라이선스 키를 ZeroPack에 입력합니다.
2-1
zpack.app 접속 후 GitHub으로 로그인

아래 버튼을 클릭해 ZeroPack을 여세요.

🔗 zpack.app 열기

"GitHub으로 로그인" 버튼을 클릭하면 GitHub 인증 화면이 열립니다.
방금 만든 GitHub 계정으로 로그인하세요.

ℹ️ "ZeroPack이 내 GitHub 계정 정보에 접근하도록 허용하시겠습니까?" 화면이 뜨면 Authorize(허용)를 클릭하세요. 안전한 과정입니다.
2-2
라이선스 키 입력

로그인 후 라이선스 키 입력창이 나타납니다.
크몽 구매 완료 후 받은 라이선스 키를 정확히 입력하고 확인을 클릭하세요.

⚠️ 라이선스 키는 공백 없이 정확히 입력하세요. 대소문자도 구분됩니다.
💡 키를 잘 모르겠으면 크몽 구매 내역 또는 판매자 메시지를 확인하세요.
3
Vercel 연동하기
Vercel은 내 홈페이지를 인터넷에 올려주는 서비스입니다. GitHub 계정으로 무료 가입 후 ZeroPack과 연결합니다.
3-1
"Vercel 연동 시작" 버튼 클릭

ZeroPack 화면에서 "Vercel 연동 시작" 버튼을 클릭하세요.
새 창(팝업)이 열리며 Vercel Integration 페이지가 나타납니다.

⚠️ 팝업이 차단됐다면? 브라우저 주소창 오른쪽에 팝업 차단 아이콘이 뜹니다. 클릭 후 "팝업 허용"을 선택하고 다시 버튼을 눌러주세요.
3-2
Vercel 계정 없으면 먼저 가입

Vercel에 계정이 없다면 "Continue with GitHub"를 클릭해 GitHub 계정으로 바로 가입하세요.
별도 비밀번호 필요 없이 GitHub 계정 하나로 연결됩니다.

💡 이미 Vercel 계정이 있다면 바로 로그인하시면 됩니다.
3-3
Integration 설치 (5단계 순서대로)

Vercel Integration 페이지에서 아래 순서대로 진행하세요:

1
"Add Integration" 버튼 클릭
2
내 Vercel 계정 선택
3
"All Projects" 선택 (권장, 이걸 선택해야 자동 배포됩니다)
4
"Install" 버튼 클릭
5
설치 완료 후 ZeroPack으로 돌아와 "연동 완료 확인" 버튼 클릭
ℹ️ "All Projects" 대신 "Specific Projects"를 선택하면 나중에 배포 오류가 날 수 있습니다. 반드시 "All Projects"를 선택하세요.
3-4
연동 완료 확인

ZeroPack 화면에 "연동 완료!" 메시지가 뜨면 성공입니다.
"다음으로" 버튼을 클릭해 4단계로 넘어가세요.

✅ Vercel 연동 완료! 이제 자동 배포 준비가 됐습니다.
4
배포 설정하기
프로젝트 이름과 사이트 언어를 설정합니다. 이름이 내 사이트 주소가 됩니다.
4-1
프로젝트 이름 입력

프로젝트 이름을 입력하세요. 이 이름이 내 사이트 주소가 됩니다.

https://내가-입력한-이름.vercel.app
⚠️ 규칙: 영문 소문자, 숫자, 하이픈(-) 만 사용 가능. 공백·한글·특수문자 불가. (예: my-cafe, john-portfolio)
ℹ️ 나중에 내 도메인(예: mycafe.com)을 연결하면 이 주소는 쓰지 않아도 됩니다. 처음엔 대충 정해도 됩니다.
4-2
배포 언어 선택

사이트가 기본으로 표시될 언어를 선택하세요.

  • 한국어 — 국내 고객 대상 사이트 (기본값 추천)
  • 영어 — 영어권 고객 또는 글로벌 사이트
💡 나중에 Visual Editor에서 내용을 바꾸면 되니, 지금은 주 사용 언어만 고르세요.
4-3
"자동 배포 시작" 버튼 클릭

설정이 끝났으면 "자동 배포 시작" 버튼을 클릭하세요.

🚫 배포가 진행되는 동안 브라우저를 닫거나 새로고침하지 마세요! 배포가 취소될 수 있습니다.
5
배포 완료 확인
자동 배포가 진행됩니다. 1~3분 정도 기다리면 내 사이트가 완성됩니다.
5-1
배포 진행 대기 (1~3분)

화면에 배포 진행 로그가 표시됩니다. ZeroPack이 다음 작업을 자동으로 수행합니다:

✓
내 GitHub에 저장소(코드) 자동 생성
✓
Vercel 프로젝트 자동 생성
✓
환경 변수 자동 설정
✓
Vercel 빌드 시작 (1~3분 소요)
🎉
배포 완료 — 내 사이트 URL 생성!
ℹ️ "사이트 버튼이 활성화됩니다 (1~3분 소요)"라는 메시지가 보이면 정상입니다. Vercel 빌드가 끝날 때까지 기다려주세요.
5-2
내 사이트 주소 확인 및 접속

배포가 완료되면 내 사이트 주소가 표시됩니다:

https://내가-입력한-이름.vercel.app

버튼을 클릭하거나 주소를 복사해 브라우저에 입력하면 내 홈페이지가 열립니다. 🎊

✅ 접속이 되면 배포 성공! 이 주소를 저장해두세요.
⚠️ 방금 배포 직후 접속 시 404 오류가 뜰 수 있습니다. 1~2분 더 기다렸다가 새로고침하면 정상 표시됩니다.
6
내용 수정하기 (Visual Editor)
배포된 사이트의 텍스트·이미지·색상을 클릭해서 직접 바꿀 수 있습니다. 코딩 불필요.
6-1
ZeroPack 대시보드에서 "편집" 클릭

ZeroPack 대시보드로 돌아가서 내 프로젝트의 "편집" 또는 "Edit" 버튼을 클릭하세요.
Visual Editor 화면이 열립니다.

6-2
텍스트 수정

바꾸고 싶은 텍스트를 클릭하면 바로 수정할 수 있습니다.
회사명, 소개글, 서비스 설명 등 모든 텍스트를 자유롭게 바꾸세요.

💡 수정한 내용은 저장 버튼을 누르면 자동으로 GitHub에 올라갑니다.
6-3
이미지 교체

이미지 위에 마우스를 올리면 교체 버튼이 나타납니다.
내 사진이나 로고 이미지를 업로드해 바꾸세요.

ℹ️ 권장 이미지 형식: JPG, PNG, WebP. 용량이 너무 크면 로딩이 느려질 수 있으니 1MB 이하 권장.
6-4
저장 → 자동 반영

"저장" 버튼을 클릭하면 변경사항이 GitHub에 자동으로 업로드됩니다.
Vercel이 이를 감지하고 1~3분 내에 실제 사이트에 자동 반영됩니다.

✅ 저장 후 1~3분 기다렸다가 내 사이트를 새로고침하면 바뀐 내용을 확인할 수 있습니다.
+
내 도메인 연결하기 (선택사항)
mycafe.com 같은 내 도메인을 연결하고 싶다면 이 단계를 진행하세요. 도메인이 없어도 vercel.app 기본 주소로 운영됩니다.
7-1
도메인 구매

가비아, 후이즈, 카페24 등에서 원하는 도메인을 구매하세요.
예) mycafe.com (연 1~3만원 수준)

💡 .com, .co.kr, .net 중 선호하는 것으로 구매하세요.
7-2
Vercel에서 도메인 추가

Vercel 대시보드 → 내 프로젝트 → Settings → Domains 탭으로 이동하세요.
구매한 도메인을 입력하고 "Add"를 클릭하면 DNS 설정 안내가 나타납니다.

7-3
DNS 설정 (도메인 구매처에서)

Vercel이 알려주는 네임서버 또는 CNAME 값을 도메인 구매처 관리 페이지에 입력하세요.
설정 반영까지 최대 24시간 소요될 수 있습니다. (보통 1~2시간 내 완료)

ℹ️ 도메인 관리 페이지 → DNS 설정(네임서버 또는 레코드 관리) → Vercel에서 안내한 값 입력
⚠
오류 발생 시 해결 방법
자주 발생하는 오류와 해결 방법입니다.
이미 동일한 이름의 저장소가 존재합니다
입력한 프로젝트 이름과 같은 GitHub 저장소가 이미 있습니다.
해결: 프로젝트 이름을 다른 이름으로 바꾸고 다시 시도하세요. (예: my-site-2)
Vercel에 동일한 이름의 프로젝트가 이미 존재합니다
Vercel에 같은 이름 프로젝트가 이미 있습니다.
해결: 프로젝트 이름을 변경하세요.
Vercel-GitHub 통합이 설치되지 않았습니다
3단계 Vercel 연동이 완료되지 않은 상태입니다.
해결: 3단계로 돌아가서 Integration 설치를 다시 진행하세요. "All Projects"를 선택했는지 확인하세요.
팝업이 차단됨
브라우저가 Vercel 연동 팝업창을 차단했습니다.
해결: 주소창 오른쪽 팝업 차단 아이콘 클릭 → "팝업 허용" 선택 → 버튼 다시 클릭
배포 완료 후 사이트 접속 시 404 오류
Vercel 빌드가 아직 진행 중입니다.
해결: 1~2분 더 기다렸다가 새로고침하세요. 최대 5분 소요될 수 있습니다.
라이선스 키 오류
키가 잘못 입력됐거나 이미 사용된 키입니다.
해결: 크몽 구매 내역에서 키를 다시 복사해 붙여넣기하세요. 공백이 들어가지 않도록 주의.

?
자주 묻는 질문
막히는 부분이 있다면 아래를 확인하세요.
GitHub, Vercel이 뭔가요? 왜 이것들을 써야 하나요?
GitHub는 내 홈페이지 파일을 저장하는 공간(창고), Vercel은 그 파일을 실제 인터넷 주소로 올려주는 서비스입니다. 둘 다 전 세계 수백만 명이 쓰는 무료 서비스이고, 이것들을 이용하기 때문에 호스팅 비용이 0원입니다. ZeroPack은 이 복잡한 연결을 자동으로 해줍니다.
GitHub, Vercel이 나중에 유료로 바뀌면 어떻게 되나요?
현재 무료 플랜으로 개인 홈페이지 운영에는 충분합니다. 혹시 유료 전환이 되더라도 내 코드는 내 GitHub에 있으므로 다른 무료 서비스(Netlify, Cloudflare Pages 등)로 이전할 수 있습니다.
수정하고 저장했는데 사이트에 반영이 안 돼요.
저장 후 Vercel이 자동으로 빌드·배포합니다. 보통 1~3분 소요됩니다. 5분이 지나도 반영이 안 되면 Vercel 대시보드에서 빌드 로그를 확인해보세요.
ZeroPack이 사라지면 내 사이트도 없어지나요?
아닙니다. 사이트는 내 GitHub + 내 Vercel 계정에서 운영됩니다. ZeroPack은 배포를 도와준 도구일 뿐이고, 배포가 완료된 사이트는 ZeroPack과 완전히 독립적으로 운영됩니다.
도메인 없이도 사이트를 운영할 수 있나요?
네. Vercel이 자동으로 "프로젝트이름.vercel.app" 주소를 무료로 제공합니다. 이 주소만으로도 충분히 운영 가능합니다. 나중에 원하면 내 도메인을 구매해 연결할 수 있습니다.
트래픽이 갑자기 많이 늘어나도 괜찮나요?
Vercel 무료 플랜은 월 100GB 대역폭이 제공됩니다. 소상공인·개인 홈페이지 수준에서는 충분합니다. 방문자가 하루 수만 명 이상 되는 대규모 서비스가 되면 그때 Vercel 유료 플랜($20/월)을 고려하시면 됩니다.
